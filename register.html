Registration flow

This is a v0.1 early-access flow. It is intentionally narrow: create a workspace request, verify it, receive an API key once, then use that key from the account console or your backend integration.

What this is and is not.

SemeAI Gate is a release-control adapter. The public business action remains SHOW, REVIEW, or BLOCK. Internally those map to PROCEED, NEEDS_REVIEW, and SILENCE. These machine values are not translated because audit and replay depend on them.

SemeAI. Generation is not release authority. Machine values remain canonical: SHOW / REVIEW / BLOCK and PROCEED / NEEDS_REVIEW / SILENCE.
