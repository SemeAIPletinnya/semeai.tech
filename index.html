SemeAI · governed release control

Generation creates a candidate. Release is a separate decision.

SemeAI is a governed release-control layer for AI-generated outputs. The model produces a candidate; the Gate evaluates evidence, policy, and context before the host receives SHOW, REVIEW, or BLOCK.

Generation is not release authority.

Why this matters

AI answers can be fluent and still not allowed.

Fake promo codes are only the smallest demo. The same boundary applies wherever generated output can mutate customer-visible facts, workflows, medical guidance, finance claims, game worlds, or internal operations.

Support
Invented entitlement
Discounts, refunds, and account promises must match business data before release.
Finance
Unsupported claim
Fees, returns, eligibility, and account status need a source of support.
Medicine
Patient-visible advice
Generated guidance should be reviewed before becoming a visible recommendation.
Worlds
Canon mutation
AI-generated lore, quests, economy, and game state should pass a release boundary.
Ops
Action escalation
Deploy, delete, config, and approval-bypass suggestions need a gate before action.

SILENCE withholds output; it does not delete the candidate, decision, or replay metadata.

SHOW

Release

Maps to PROCEED. The answer is supported enough to be shown automatically.

REVIEW

Hold for operator

Maps to NEEDS_REVIEW. The system keeps audit metadata and does not auto-release.

BLOCK

Withhold

Maps to SILENCE. Release is denied, execution is withheld, audit is preserved.

Deployment model

SaaS when speed matters. Self-hosted when data cannot leave.

The architecture supports three paths: open-source adapter, hosted API, and enterprise self-hosted deployments for teams that must keep documents inside their own perimeter.

Open SourceRun the adapter yourself. Inspect tests, examples, schemas, and receipts. Self-hosted EnterpriseDeploy in your VPC or internal network. We do not need to see your documents. SaaS APIFast pilot path with workspace registration, live checks, API keys, and account console.
Trust boundary

The gate runs where your data is allowed to live.

SemeAI is not forced into one deployment shape. Small teams can start with the hosted API. Sensitive teams can run the same contract inside their own perimeter, so documents do not have to leave.

Public instrument

See the release principle operate on a real repository.

The Repository Evidence Benchmark reads bounded public GitHub evidence and produces inspectable criteria, a candidate score, a display-only Benchmark Presentation Gate decision, and a local receipt.

SHOW means evidence is sufficient to display. It is not a quality approval, security verdict, or production endorsement.
