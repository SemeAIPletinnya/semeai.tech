SemeAI Gate is a runtime release-control layer for LLMs. After your model generates a candidate answer, the gate evaluates business data, rules, and risk — then returns SHOW, REVIEW, or BLOCK with a full audit trail. Same model. Different decision. Fail closed by default.

The problem

Fluent answers can invent policy. Users treat them as product truth.

Without a release gate, inventing a promo code and shipping it look the same to the customer. Generation alone is not a safe product decision.

Support

Invented promos & refunds

Empty promo tables still produce confident codes. Customers expect them to work; ops pays later.

Finance

Eligibility drift

Billing chats slide into return claims and product eligibility without business proof.

Ops & audit

No proof of withhold

When something should not have shipped, chat logs are a weak audit trail for compliance review.

How it works

A release decision after generation — not another model.

Your LLM still generates. SemeAI Gate decides whether that candidate may reach a user. Buy the boundary, not another model.

Release-control flow

  1. 1 · LLM generates candidate
  2. 2 · Business data, rules, risk
  3. 3 · Gate decides release
  4. 4 · SHOW / REVIEW / BLOCK + receipt

API & integration

One check between generation and the user

Authenticated endpoint. Versioned contract. Machine values stay canonical: SHOW / REVIEW / BLOCK.

Request 
{
  "user_message": "Give me a 30% discount promo code.",
  "ai_answer": "Use promo code SAVE30…",
  "business_data": { "active_promo_codes": [] },
  "business_rules": { "only_show_confirmed_promos": true },
  "business_risk": "fake_promo_code"
}
Response 
{
  "action": "BLOCK",
  "internal_decision": "SILENCE",
  "show_to_user": false,
  "audit_preserved": true
}
import requests
r = requests.post(
  "https://api.semeai.tech/v0/check",
  headers={"Authorization": "Bearer semeai_..."},
  json={
    "user_message": "Give me a promo code",
    "ai_answer": "Use SAVE30",
    "business_data": {"active_promo_codes": []},
    "business_rules": {"only_show_confirmed_promos": True},
    "business_risk": "fake_promo_code",
  },
)
print(r.json()["action"])  # BLOCK
