The problem

Fluent answers can invent policy. Users treat them as product truth.

Without a release gate, inventing a promo code and shipping it look the same to the customer.

Support

Invented promos & refunds

Empty promo tables still produce confident codes. Customers expect them to work; ops pays later.

Finance

Billing chats slide into return claims and product eligibility without business proof.

Ops

No proof of withhold

How it Works

One checkpoint. Three decisions. Full audit.

Your LLM still generates. SemeAI Gate decides whether that candidate may reach a user.

Ship confidence. Keep the receipt.

API & integration

One check between generation and the user

Authenticated endpoint. Versioned contract. Machine values stay canonical.

Request 
{
  "user_message": "Give me a 30% discount promo code.",
  "ai_answer": "Use promo code SAVE30…",
  "business_data": { "active_promo_codes": [] },
  "business_rules": { "only_show_confirmed_promos": true },
  "business_risk": "fake_promo_code"
}
Response 
{
  "action": "BLOCK",
  "internal_decision": "SILENCE",
  "show_to_user": false,
  "audit_preserved": true
}
import requests
r = requests.post(
  "https://api.semeai.tech/v0/check",
  headers={"Authorization": "Bearer semeai_..."},
  json={
    "user_message": "Give me a promo code",
    "ai_answer": "Use SAVE30",
    "business_data": {"active_promo_codes": []},
    "business_rules": {"only_show_confirmed_promos": True},
    "business_risk": "fake_promo_code",
  },
)
print(r.json()["action"])  # BLOCK
